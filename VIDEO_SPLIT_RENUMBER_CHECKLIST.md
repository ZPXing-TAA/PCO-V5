# Video Split Renumber Checklist

## 目标

- 录制完成后做后处理
- 按 `route + label` 重新连续编号
- 保持当前目录命名风格：`natlan_rXX_<label>NN`
- 不使用 `part1` / `part2`
- 不改 route 语义，不改运行时动作执行

## 处理时机

- 不是所有动作都录完再统一处理
- 是单个原始录制段目录保存完成后，立刻处理
- 但在开始处理前，必须先根据 route 定义预计算好该 route 的最终编号计划

## 编号规则

- 编号范围只在同一个 `route + label` 内
- 同 route 下不同 label 互不影响
- 不同 route 互不影响
- 单个原始段展开出的多个最终片段直接占用多个连续编号
- 同 label 的后续原始段整体顺延

## 切分规则

- 只处理 `record_start` 到 `record_stop` 的录制段
- 先按 route 声明时长决定段数，不按实际文件时长决定段数
- 若 `t_route < 5`，保留原视频，输出 `1` 段
- 若 `t_route >= 5`，只输出从开头开始的整 `5s` 计划片段
- 最终片段数为 `floor(t_route / 5)`
- route 规则决定丢弃的最后不足 `5s` 余数继续丢弃
- 若实际文件只比计划终点略短，例如 `10s` 实录 `9.8s`，最后一个计划片段直接截到文件结尾，不因此少一段
- 不做均分
- 不保留余数

## 例子

原始：

```text
run01   <- 10s
run02   <- 6s
run03   <- 5s
```

处理后：

```text
run01   <- 原 run01 的前 5s
run02   <- 原 run01 的后 5s
run03   <- 原 run02 的前 5s
run04   <- 原 run03 的 5s
```

补充：

- 如果原 `run01` 实际只有 `9.8s`，仍然输出 `run01` 和 `run02`
- 其中 `run02` 变成 `5.0s ~ 9.8s`

## 输入

- 路线定义：`routes/natlan/*.py`
- 原始录制目录：

```text
<video_base>/<label>/natlan_rXX_<label>NN/<config_id>.mp4
```

## 实现顺序

1. 读取 route，提取原始录制段列表
2. 计算每个原始段最终需要展开成几段
3. 规则固定为：
   - `t_route < 5` -> `1`
   - `t_route >= 5` -> `floor(t_route / 5)`
4. 为每个 `route + label` 预计算最终编号映射
5. 单个原始录制段目录一保存完成，就按映射执行
6. 读取每个实际视频文件的真实时长 `t_media`
7. route 声明时长 `< 5` 的段：直接写入最终目录
8. route 声明时长 `>= 5` 的段：按计划切连续 `5s` 窗口
9. 对最后一个计划片段，结束时间取 `min(计划结束点, t_media)`
10. route 规则决定丢弃的不足 `5s` 余数继续丢弃
11. 不保留旧的临时命名

## 工程要求

- 先做 dry-run
- dry-run 至少打印：
- 原始目录
- route 声明时长
- 原始时长
- 最终片段数
- 是否会按 route 规则截掉尾部
- 是否存在实际短录
- 切分后目标目录
- 最终编号映射
- 实际执行时优先输出到新目录，避免原地覆盖

## 验收

- `64 -> 90` 数量正确
- 所有最终片段都满足：
- 原始 `<5s` 的段原样保留
- 原始 `>=5s` 的段只输出整 `5s` 片段
- 若实际文件轻微短录，最后一个计划片段可以略短，但不能因此少一段
- 同 `route + label` 编号连续，无重复、无跳号
- 最终目录中仍然使用原始 `config_id.mp4` 文件名
